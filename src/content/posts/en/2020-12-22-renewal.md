I changed the structure of this site to the following format.

|  | Framework | Hosting |
| --- | --- | --- |
| Old | [Gatsby.js](https://www.gatsbyjs.com/) | [Netlify](https://www.netlify.com/) |
| New | [Next.js](https://nextjs.org/) | [Vercel](https://vercel.com/) |

When I built it using Gatsby.js in April this year, I managed to get it running, but I did so without understanding React.js (or rather, Gatsby.js) at all.

Eventually, I was just doing my best to keep things working without really understanding them, and I thought I would have a hard time if I wanted to fix styles or make changes in the future.

I built it, but because I was staying indoors more than usual due to the COVID-19 pandemic and feeling mentally exhausted, I ran out of blog topics and ended up leaving it alone.

During that time, I started to feel the momentum behind Next.js—to the point where even a frontend novice like me could tell it would be a good choice. Having also had the opportunity to touch React personally, I decided to do a renewal using Next.js and Vercel this time.

For more details, please check the GitHub repository, but the technical stack is as follows:

- Framework: Next.js
- Hosting: Vercel
- CSS Framework: tailwind.css
    - I'm using [Twin](https://github.com/ben-rogerson/twin.macro) to use it with Styled Components

Posts themselves are written in Markdown (GFM), and I've enabled code highlighting and LaTeX support.

I want to build a habit of continuous output, so I will do my best.

Learning from my previous experience, I know that if I cram too much into a single post, I'll run out of things to write about and end up quitting. Therefore, I plan to stock up on topics and release them in small increments.
